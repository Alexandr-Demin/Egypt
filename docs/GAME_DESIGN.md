# EGYPT — Game Design Document

> Status: **v0.1 spec (MVP)** · pixel-art slide-maze · setting: Ancient Egypt
> Keep this in sync with the code.

---

## 1. Concept (pitch)

**EGYPT** — аркадный лабиринт «в один свайп»: герой скользит по плитам гробницы
до ближайшей стены, собирает золото, обходит ловушки и стражей и доходит до
выхода-саркофага. Мгновенное управление, «ещё один заход», короткие
напряжённые забеги. Сеттинг — Древний Египет:
пирамиды, гробницы, иероглифы, скарабеи, мумии, Анубис.

**Жанр:** hyper/hybrid-casual arcade maze.
**Фишка:** инерционное скольжение (slide-until-wall) + читаемый пиксель-арт.
**Сессия:** 20–60 сек на уровень, мгновенный рестарт.

---

## 2. Core loop

- **Moment-to-moment:** свайп/стрелка → герой скользит по прямой до стены,
  собирая всё на пути. Сталкиваешься со стражем/ловушкой → смерть → рестарт.
- **Уровень:** добраться от старта до выхода-саркофага. Золото — за очки/звёзды.
- **Сессия (v0.1):** Play → Уровень 1 → Уровень 2 → экран победы → меню.
- **Мета (позже, не в v0.1):** звёзды за уровни, разблокировка масок/скинов,
  endless-режим «погружение в гробницу».

---

## 3. Управление

- **Тач:** свайп в 4 направлениях (вверх/вниз/влево/вправо).
- **Десктоп (для отладки/веб):** стрелки и WASD.
- Один свайп = одно скольжение до препятствия. Ввод во время скольжения
  буферизуется и срабатывает у стены. Без диагоналей.

---

## 4. Сеттинг и арт-дирекшн

**Стиль:** пиксель-арт. Чёткие пиксели, nearest-neighbor масштабирование,
без сглаживания.

**Тех-спек пикселя:**
- Тайл: **16×16 px** (логический), сетка уровня ~**11×13** тайлов.
- Внутренний рендер-буфер кратен 16; апскейл к экрану целым множителем
  (integer scaling), `image-rendering: pixelated`, `ctx.imageSmoothingEnabled=false`.
- Портретная ориентация, поле по центру, рамка-камень вокруг.

**Палитра «Проклятие» (багровое свечение + золото на тёмном):**
Источник истины — `PAL` в `src/sprites.js`. Стены — чёрные, виден только
светящийся точечный контур границ; пол — тот же тёмный фон.
| Роль | Ключ | Hex |
|---|---|---|
| Фон / пустота | bg | `#0c0406` |
| Контур стен (база) | wall | `#d23a2a` |
| Контур (свет) | wallHi | `#ff7a5c` |
| Контур (яркий край) | wallEdge | `#ffc0ad` |
| Золото / сбор / герой | gold | `#ffd21e` |
| Золото (блик) | goldHi | `#fff6b0` |
| Бинты мумии | wrap | `#efe2cf` |
| Кровь / опасность | red | `#ff4030` |

**Объекты:**
| Роль | EGYPT-объект | Спрайт |
|---|---|---|
| Дата-спрайт (игрок) | **Мини-Анубис** — юный бог с головой шакала | 16×16, чёрно-золотой, голубой анх в руке |
| Биты данных (сбор) | **Золотые анхи / монеты** | мерцающий 8×8 |
| Сентинел (враг) | **Мумия** (патруль) / **Скарабей-рой** | 16×16, красные глаза |
| Шип-разряд (ловушка) | **Шипы / дротиковая ловушка** | выезжают из плиты |
| Портал-выход | **Саркофаг / дверь с иероглифами** | пульсирующий лазурит |
| Стена | **Каменный блок** с иероглифом | 16×16 тайл |

> **Герой — мини-Анубис** (зафиксировано). Враги намеренно НЕ Анубисы
> (мумии/скарабеи), чтобы не путать игрока с собой. Опц. альт-скины позже:
> маска-расхититель, скарабей.

## 5. v0.1 — точный скоуп (MVP)

Только три экрана. Ничего лишнего.

### 5.1 Экран TITLE (меню)
- Фон: гробница/пирамида в пиксель-арте, лёгкий параллакс песка (опц.).
- Логотип **EGYPT** (пиксель-шрифт, золото).
- Кнопка **PLAY** (крупная, по центру-низу) → запускает Уровень 1.
- (Опц., если дёшево) маленький переключатель звука 🔊 в углу.
- Без настроек/магазина/лидерборда — это потом.

### 5.2 Уровень 1 — «Первый зал» (обучение через дизайн)
- Открытый зал с парой колонн-стен — игрок интуитивно понимает скольжение.
- Только сбор золота + выход-саркофаг. **Без врагов и ловушек.**
- Цель: дойти до саркофага. Золото — бонус.
- Подсказка-плашка: «Свайп — скользи до стены» (исчезает после 1-го хода).

### 5.3 Уровень 2 — «Зал стража»
- Лабиринт плотнее (колонны/коридоры).
- **1 мумия** (горизонтальный патруль) + **1 шип-ловушка**.
- Смерть от мумии/шипа → рестарт уровня (3 жизни ♥).
- Цель: дойти до саркофага живым.

### 5.4 Поток v0.1
```
TITLE ──PLAY──▶ Level 1 ──exit──▶ Level 2 ──exit──▶ WIN
  ▲                                                  │
  └──────────────── «В меню» ◀───────────────────────┘
```
- На WIN: «Гробница пройдена», суммарное золото, кнопки **Заново** / **В меню**.
- Смерть при 0 жизнях → экран «Проклятие настигло» → Заново / В меню.

`#` стена · `.` золото · ` ` пол · `P` старт · `E` выход(саркофаг) · `X` мумия(гор.) · `^` шипы

## 6. Прогресс и экономика (v0.1)
- **Жизни:** 3 ♥ на прохождение, рестарт уровня при смерти.
- **Золото:** очки за забег, показывается на WIN. Без траты в v0.1.
- **Сохранение:** в `localStorage` под `CONFIG.saveKey` — лучший результат
  и пройденность (`progress.best`, `progress.level`).
- Звёзды/маски/IAP — вне v0.1 (см. §10).

## 7. Монетизация
- v0.1 — без монетизации (фокус на feel).
- Позже: hybrid-casual (реклама на рестарт/континуе + IAP на скины-маски).

## 8. Аудио
- Пиксель-чиптюн: шаг-скольжение, сбор золота (звонкий), смерть (низкий),
  фанфара на выход. Тумблер звука (`src/sound.js` уже умеет on/off + persist).
- v0.1: достаточно 4 синтезированных тона из `sound.js` (`tap/win/lose`).

---

## 9. Tech — привязка к каркасу

- Vanilla JS PWA, **canvas** рендер, без сборки (как весь проект).
- Пиксель: один offscreen-буфер 16-кратный, integer-апскейл в основной canvas.
- Состояние: `src/state.js` (`progress`, `stats`, `settings`).
- Файловый план под игру (добавить к скелету):
  | Файл | Назначение |
  |---|---|
  | `src/game.js` | игровой цикл, скольжение, коллизии, камера |
  | `src/levels.js` | карты уровней (ASCII → grid) |
  | `src/sprites.js` | пиксель-спрайты (data-URI / процедурные) + палитра |
  | `src/screens.js` | TITLE / WIN / GAMEOVER экраны |
  | `src/main.js` | роутинг экранов + boot (уже есть) |
- Cache-buster `?v=…` на всех импортах == `CONFIG.build` (см. CLAUDE.md).

## 10. Roadmap (после v0.1)
1. Ещё уровни + «миры» (Гробница → Пирамида → Храм Анубиса).
2. Новые враги: дротиковые турели, преследователь-скарабей, каменный голем.
3. Endless «погружение»: авто-скролл вниз, наползающий песок/лава.
4. Звёзды за уровень (за все золото / без смертей / на скорость).
5. Косметика: маски-скины игрока (IAP), сезонные ивенты.
6. Лидерборд + AppMetrica-события (см. `docs/ANALYTICS.md`).

## 12. Решения (зафиксированы)
- ✅ **Игрок — мини-Анубис.** Враги — мумии/скарабеи (не Анубисы).
- ✅ **Ассеты процедурные** — спрайты и шрифт рисуются в коде (canvas /
  data-URI), без внешних файлов. Шрифт: процедурный 5×7 спрайт-шрифт под
  цифры/латиницу. Спрайты: пиксель-карты в `src/sprites.js`.
- ✅ **Карты — хардкод** ASCII в `src/levels.js`, без редактора в v0.1.
